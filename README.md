# Reddit Multi-Community Trend Analysis

Mini-Project-3 for Data Engineering & Processing (DEP). This project compares Reddit search results across 14 subreddits for 150 curated queries, measures cross-community overlap, and trains regression models to predict inter-subreddit agreement from query features.

## Overview

The pipeline searches Reddit's public JSON API for 150 queries across five categories: `tech`, `news`, `health`, `howto`, and `science`. For every query, it collects up to the top 50 posts from each subreddit, stores raw results in MongoDB, computes pairwise overlap metrics for all 91 subreddit pairs, trains machine-learning models, and generates both static Plotly outputs and a React dashboard.

No Reddit API key is required. Requests use unauthenticated public Reddit JSON reads with a project User-Agent and a 1.5 second delay between successful requests.

## Subreddits

The analysis covers:

`science`, `technology`, `worldnews`, `news`, `todayilearned`, `askscience`, `programming`, `explainlikeimfive`, `Futurology`, `environment`, `learnprogramming`, `datascience`, `economics`, and `history`.

`r/worldpolitics` is intentionally excluded because it had a 100% fetch failure rate.

## Metrics

For each query and subreddit pair, the project computes:

- `Jaccard@5`
- `Jaccard@10`
- `Jaccard@TOP_K`, stored as `jaccard`
- Spearman rank correlation, stored as `NaN` when fewer than three URLs overlap
- Rank-biased overlap (RBO), using `p=0.9`

URLs are normalized by lowercasing, removing `www.`, and stripping trailing slashes to reduce false mismatches.

## Machine Learning

The prediction target is `Jaccard@10`. Models are trained per subreddit pair using eight query features:

- `query_len`
- `word_count`
- `has_question`
- `category_enc`
- `is_navigational`
- `avg_word_len`
- `has_number`
- `unique_char_ratio`

Models:

- Linear Regression
- Ridge with `GridSearchCV`
- Random Forest with `GridSearchCV`
- Gradient Boosting with `GridSearchCV`
- SVR with `StandardScaler` pipeline and `GridSearchCV`
- Mean baseline with `DummyRegressor`

The pipeline stores 70/30 split metrics, 5-fold CV MAE, best parameters, permutation/tree feature importances, predictions, and bootstrap 95% confidence intervals for mean `Jaccard@10`.

## Setup

Start MongoDB:

```bash
docker compose up -d mongo
```

Activate the DEP virtual environment and install Python dependencies:

```bash
source ../../dep/bin/activate
uv pip install -r requirements.txt
```

Create the local environment file:

```bash
cp .env.example .env
```

Default environment values:

```text
MONGO_URI=mongodb://localhost:27017
MONGO_DB=dep_mp3
TOP_K=50
```

## Run the Analysis

Fresh run with Reddit fetching:

```bash
python analysis.py --force-refresh
```

Reuse cached MongoDB results and regenerate metrics/models/outputs:

```bash
python analysis.py
```

Optional recalibration step:

```bash
python recalibrate.py
```

`recalibrate.py` is intentionally gitignored and should be run only after `analysis.py` when the project needs the domain-prior adjusted metrics and retrained model artifacts.

## Outputs

Generated outputs include:

- `insights.txt`: text summary of result coverage, overlap metrics, model leaderboard, failures, and data quality.
